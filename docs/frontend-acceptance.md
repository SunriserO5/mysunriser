# 前端最小验收说明

## 1. 启动方式

### 启动后端

在项目根目录执行：

```bash
cd backend
./mvnw spring-boot:run
```

默认监听 `http://localhost:8080`。

### 启动前端开发服务器

在项目根目录执行：

```bash
cd frontend
npm run dev
```

默认访问 `http://localhost:5173`。

## 2. 联调验证路径

1. 首页：`/`
2. 博客列表：`/blog`
3. 文章详情：`/blog/:slug`
4. 关于页：`/about`
5. 登录页：`/login`
6. 管理员新建文章：`/admin/posts/new`
7. 管理员账号管理：`/admin/users`

## 3. 关键交互验收点

1. 列表页分页可切换，URL 会保留 `?page=` 参数。
2. 切换分页后自动滚动到页面顶部。
3. 列表页加载失败时展示“重试”按钮。
4. 详情页加载失败时展示“重试”按钮。
5. 文章详情页标题会动态更新为 `文章标题 | MySunriser`。
6. 首页主按钮和右上角导航在正常态与悬停态均应保持可读。
7. 未登录或普通用户在 `/blog` 看不到“新建文章”按钮。
8. admin 在 `/blog` 能看到“新建文章”按钮，按钮文字与底色保持可读。
9. admin 点击“新建文章”进入 `/admin/posts/new`，填写 slug、标题、正文、状态后可保存并跳转到 `/blog/{slug}`。
10. 新建文章时重复 slug 应展示后端错误，不覆盖原文章。
11. admin 打开 `/blog/:slug` 能看到“编辑”按钮，编辑标题、正文、状态、发布时间后可保存并回到阅读态。
12. 新建/编辑表单点击“取消”不提交变更，分别返回列表页或阅读态。

## 4. 已知限制

1. `npm run preview` 默认不代理 `/api`，仅用于预览构建产物。
2. 异常“超时”场景依赖网络和后端行为，本地可通过断开后端服务模拟。
3. 文章摘要 `summary` 暂未纳入新建/编辑表单。
